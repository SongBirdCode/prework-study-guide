HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The base element defines the base URL for a webpage.
  • The script element embeds client-side scripts in the HTML document.
  • HTML's role in web development is to define the framework for the page.
  • The style element is used to embed CSS and contains the CSS rules needed to style the page.
  • The link element defines the relationship between the current page and an external source.
  • The meta element defines metadata for the page.
  • The script element is used to embed JavaScript on the page.
  • The noscript element defines the content to be displayed if scripts are disabled.
  • Semantic elements are often used to divide a page.
  • HTML attributes are used to provide additional information about an element.
  • There are global and non-global attributes which indicate whether they are used with specific element attributes or not.

CSS

  • The style element defines the CSS styles associated with a website.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The border property assigns a border with a width of 5px that is solid and gray.
  • The box-shadow CSS property adds shadow effects around an element's frame.
  • Selector: HTML element name at the start of the ruleset.
  • Declaration: A single line rule that specifies which of the element's properties to style.
  • Properties: These are ways to style an HTML element.
  • To the right, after the colon, it is called the property value.
  • Property value chooses one out of many possibile appearances for a given property.
  • CSS layout is based on the box model where each box takes up space.

Git

  • There are three main states files can reside: modified, staged, and committed.
  • Modified means file has changed but not commmited in database.
  • Staged means a modified file is marked to go into commit snapshot.
  • Commited means that the data is safely stored in local database.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Obtaining a Git repository can happen by cloning or take a local directory and turn it into a Git repository.
  • Tracked files are files that were in the last snapshot as well as newly staged files.
  • Tracked files can be unmodified, modified, or staged.
  • git add: begins to track a new file.

JavaScript

  • JavaScript is a powerful programming language that can add interactivity to a website.
  • Variables are containers that store values.
  • To declare a variable use the keyword, let.
  • After declaring a variable you can give it a value.
  • A sequence of text enclosed in single of double quote is called a string.
  • Numbers do not have quotes around them.
  • A True/False value is a Boolean variable.
  • The words true and false in a Boolean variable do not need quote marks.
  • An Array is a structure that allows you to store multiple values in a single reference.
  • An Object can be anything stored in a variable within JavaScript.
  • Values within variables hold different data types.
